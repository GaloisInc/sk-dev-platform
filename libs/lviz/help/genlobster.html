Shrimp to Lobster Compilation (gen-lobster)

Navigation: Home > gen-lobster

Purpose

gen-lobster is the reverse compiler for policy DSL: it takes Shrimp language code and builds up hierarchy to create a lobster DSL file. This file can be edited, viewed, information flow checked, and then the result can be compiled back into Shrimp format which can easily be converted into SELinux.  The reverse compilation step is shown relative to the other tools in Figure 1.

Running Gen-Lobster

genLobster [global options] <input directory>
  Generate lobster policy module(s) from selinux policy in <input directory>
Global options:
--single <file>    Place all lobster definitions in a single <file>
--multiple <dir>   Generate multiple lobster files and place them in <dir>
--ifdefs <file>    Read ifdef declarations from <file>
--kind-errors      Output errors generated by kind checking in shrimp. Default is no errors.
--infer-missing    infer missing classes (suitable for graphing by lviz) One class is generated for each referenced class that is not defined and one domain placement is created for each class not placed. This simplifies viewing in LViz because created classes only have ports that are defined, not all possible ports defined - which can be a large number with classes File( ) and Process( )

Examples:
genLobster policy --single lobster_file.lsr --infer-missing

genLobster --multiple dir --kind-errors policy

Limitations

There are always multiple ways to define hierarchy relative to a complex system.  The intent of Gen-Lobster is to model the implied hierarchy already found in the reference policy by using similar names and structures.  Ad-hoc hierarchy extraction is not the focus of this version of the tool. 
Also, reverse compilation is an approximation.  The current focus is to capture the type-enforcement aspect of the SELinux policy (type declarations and access vector rules).
