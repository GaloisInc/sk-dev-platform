Overview & Goals

Many security policy languages are difficult to use because they require programming large sets of low level rules to be effective.  SELinux is an example policy language which can require 100,000's of rules to effective secure an operating system.  Because of the complexity, modifying SELinux rules to allow for any change presents a large barrier.  Further, programming a system composed of multiple different policy elements with different types of rules (SELinux, Xen, guard) can be more daunting.  An example system might be two SELinux servers, a virtual machine, and an external guard.

The Policy DSL project goal was to provide tools and techniques to allow for better handling of security policy creation and analysis.  The initial focus was to provide tools targeted at solving SELinux configuration.  Later work created a more general approach to solving policy problems which could be applied across different policy languages, including the Lobster DSL.

Lobster DSL  is a high level policy specification language which provides methods of abstracting, understanding, and analyzing security policies.  The Lobster Tool has the ability to analyze the DSL, compile it into Shrimp DSL format (a subset of SE Linux Native), test data flow assertion (using Symbion extensions to the DSL), and display  the policy graphically.

Workflow and Tool Interaction

Policy DSL tools are anchored around two languages: Lobster, the abstract DSL used to describe any policy and Shrimp DSL, which is close to the native SELinux policy with some additional checking added.  The tools available are shown in Figure 1.


Figure 1: Overview of the Policy DSL Tool Set

The workflows supported by this set of tools are based around creating and analyzing policies.  Policies can be created at an abstract level via Lobster and lower level polices can be generated from them. Alternatively existing SELinux policies and be analyzed or abstracted into Lobster.  Some example workflows:
  1. Creating a Lobster policy in a text file and viewing it in LViz.  Symbion and the compilation checking can be run from LViz
  2. Compile a Lobster policy, execute any Symbion assertions, and generate Shrimp DSL
  3. Running genLobster to reverse compile Shrimp files into Lobster DSL
  4. Running Shrimp tools on a policy to identify format and style problems
  5. Viewing the Reference policy in Slide and seeing Shrimp "lint" like problems associated with it.

Tools & Components

The tools and languages each have their own specific pages covering them in more detail.   Links to each tool with a short description follows:
  • Lobster DSL - Summary of the Lobster Domain Specific Language used to define policies in an abstract fashion.  It is not tied to any specific policy languages and can be used as the starting point for any policy definition. Compilation to multiple policy languages is possible.
  • Symbion - Symbion is an extension to the Lobster DSL language that consist of a series of assertion statements about information flow in the policy.  Symbion is run as part of compilation of the policy and can be run from LViz, the lobster policy visualization tool.
  • LViz, - Lobster Visualization tool which will display a lobster policy.  Symbion checks can be viewed as well.
  • Lobster compiler- Overview of the Lobster compiler which current converts a lobster policy into a shrimp policy. 
  • Shrimp  - Shrimp is a format and tool set for analyzing SELinux policies. 
  • GenLobster, - Reverse compiler which takes SELinux like Shrimp and converts it to Lobster
  • Slide - Viewing tool for examining SELinux policies while showing Shrimp "lint" like issues.

Supported Systems

Policy DSL tools were designed to operate on Linux OS, although most of the testing has been on Ubuntu and Centos Linux.
The SLIDE program from Tresys is tested and designed around Fedora Linux because it uses Fedora's version of Eclipse. 

Technical Papers

Multiple technical papers have been produced during the Policy DSL project which are more detailed reference material.  A few of the details of the languages have been modified since the papers have been written, but the underlying concepts are the same.  This set of help files provide the latest up to date syntax of the tools and languages.
The papers include:
  • Policy-DSL-2009 A summary of the Policy DSL work, covering shrimp, symbion and lobster.  This is the most up to date paper
  • Policy DSL Slides - presentation providing overview of Policy DSL work.
  • Lobster DSL paper describes the lobster generic policy language and relates it to SELinux.
  • Symbion paper describing the original intentions of the symbion language.  The implemented version is slightly different than the paper and is properly described below.
  • SELinux Policy Semantics: Describes semantics of SELinux and the associated Haskell implementation of the semantics.
  • Shrimp Mar 2008 Presentation: A discussion of the Shrimp approach to analyzing the reference policy
